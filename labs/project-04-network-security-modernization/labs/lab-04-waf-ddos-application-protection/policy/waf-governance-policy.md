# WAF Governance Policy

## Purpose
Ensure WAF rules are managed, reviewed, and aligned to business risk.

## Governance Principles
- Rules must map to a defined threat
- Changes require justification and owner
- Rules reviewed quarterly
- Emergency rules expire unless renewed

## Roles
- Security Engineering: Rule implementation
- App Owners: Business impact validation
- Risk/GRC: Oversight and acceptance

## Metrics
- False positive rate
- Attack mitigation success
- Rule age and coverage

Prove Which rule classes block:

- SQLi

- XSS

- Path traversal

- API abuse

Evidence of:

- Blocked attacks

- Allowed legitimate traffic

- Why default rules are not enough
